# Política de Seguridad

## Reportando Vulnerabilidades

**NO uses issues públicos para reportar vulnerabilidades de seguridad**

Si encuentras una vulnerabilidad de seguridad, por favor:
1. Envía un correo a: [manasesdiaz67@gmail.com]
2. Describe la vulnerabilidad en detalle
3. Incluye pasos para reproducir
4. Espera nuestra respuesta (48 horas máximo)

## Versiones Soportadas

| Versión | Soporte          |
|---------|------------------|
| 1.0.x   | Soporte completo |

## Mejores Prácticas de Seguridad

### Para Usuarios:
1. Siempre ejecutar como administrador
2. Verificar hash de los archivos
3. Crear punto de restauración antes
4. Tener antivirus actualizado

### Para Desarrolladores:
1. Validar todas las entradas
2. Usar rutas absolutas con cuidado
3. Implementar rollback en errores
4. Loggear todas las operaciones

## Actualizaciones de Seguridad
Las actualizaciones críticas de seguridad serán publicadas:
- Dentro de 72 horas de descubrimiento
- Con parches disponibles
- Con documentación de cambios

## Responsabilidad
Estos scripts son proporcionados "como están". Los usuarios son responsables de:
- Hacer backups antes de usar
- Verificar compatibilidad con su sistema
- Supervisar el proceso de optimización
